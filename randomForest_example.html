Trees and Forests¶

The examples below are meant to give a high level introduction to the use of decision trees and random forests. The article that we looked at uses Random Forests to determine information about the appropriate color for a map. These are quantitative questions and lead to the use of a regression tree. Despite the use of categorical problems and hence, use classification trees, the libraries in R discriminate between these questions on their own, and the implementation is consistent.

Let's again consider the introductory paragraph from Mapping Coral Reef Resilience Indicators Using Field and Remotely Sensed Data.

Two types of predictive model were used to produce maps of each resilience indicator: random forest [69] and universal kriging [70] (Figure 2). Random forest is a non-parametric ensemble classifier that predicts the value of a single response variable from the values of multiple predictors; this model type has previously been shown to outperform other regression model types for modelling ecosystem variables in coral reef environments [47,71]. A random forest model is composed of multiple regression trees [72], where splits at each node in each tree are based on a random subset of the available predictors [69]. For each resilience indicator, a model was trained on a data set consisting of the observed values of the indicator at the field sites (the response) and the values of all environmental data layers at the field site locations (the predictors). Model development was done using R [73], using the randomForest package [74]. Default values were used for random forest parameters, except for the number of trees in each forest and the number of predictors used to partition the data at each node. These two parameters were optimized with internal 5-fold cross-validation for each training partition, where the combination producing the lowest error value, quantified as the root mean square error (RMSE), within the training set was used.

The scientists used data from field stations to generalize and produce maps as shown below. The big idea is how do you use multiple measures to classify some unknown measurement -- whether this is categorical or numeric. You can see that the colors are determined by a range of values. These ranges outside of the dots for the field stations have been determined by the randomForest and universal krigging models.

Random Forest's¶

The notion of the decision tree is straightforward, however by using a single tree we can run into the problem of overfitting a model and losing predictive power. A Random Forest builds a bunch of trees and uses the results across them to determine the model. Rather than visualizing a tree itself, we will look at the results of all the trees being averaged to understand the variable importance.

Below, we can examine the iris dataset that contains measurements from three different species of flowers. We could ask which of these measurements is a better predictor of species. Here, we can compare the results of a decision tree to that of forest of trees. We will first load the randomForest library, and chop the iris data into two pieces: the quantitative measurements and the species categories.
